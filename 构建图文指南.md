# 📱 邓恩赐原生手机 APP - GitHub 构建图文指南

## 🎯 快速开始

你的项目已经初始化完成！现在只需要 3 个步骤：

---

## 第 1 步：创建 GitHub 仓库（2 分钟）

### 1.1 打开 GitHub
浏览器已自动打开：https://github.com/new

或者手动访问：
- 确保已登录 GitHub（没有账号需要先注册）
- 点击右上角 `+` → `New repository`

### 1.2 填写仓库信息
```
Repository name*: dengenci-native-app
Description*: 邓恩赐原生手机 APP
```

### 1.3 设置选项
- ✅ Public (公开)
- ❌ 不要勾选 "Add a README file"
- ❌ 不要勾选 "Add .gitignore"
- ❌ 不要勾选 "Choose a license"

### 1.4 创建
点击绿色按钮：**Create repository**

---

## 第 2 步：推送代码（1 分钟）

GitHub 创建仓库后，会显示快速设置页面。

### 2.1 复制你的仓库地址
在页面找到：
```
…or push an existing repository from the command line
```

复制下面的 HTTPS 地址（类似）：
```
https://github.com/你的用户名/dengenci-native-app.git
```

### 2.2 在终端执行命令
打开终端，已自动定位到项目目录，依次执行：

```bash
# 添加远程仓库（把下面的地址替换成你的）
git remote add origin https://github.com/你的用户名/dengenci-native-app.git

# 设置主分支
git branch -M main

# 推送代码
git push -u origin main
```

**如果提示登录：**
- 用户名：输入你的 GitHub 用户名
- 密码：**不是** GitHub 密码，而是 Personal Access Token
  - 创建 Token：https://github.com/settings/tokens/new
  - 勾选 `repo` 权限
  - 点击 "Generate token"
  - 复制 token 粘贴到密码位置

---

## 第 3 步：下载 APK（5-10 分钟）

### 3.1 查看构建进度
1. 访问你的 GitHub 仓库
2. 点击顶部的 **Actions** 标签
3. 会看到 "Build Android APK" 正在运行
4. 等待约 5-10 分钟，状态变成绿色 ✅

### 3.2 下载 APK
1. 点击最新的构建记录（顶部那个）
2. 滚动到页面底部
3. 在 **Artifacts** 区域
4. 点击 **dengenci-android-debug** 下载
5. 解压 zip 文件
6. 得到 `app-debug.apk`

---

## 📲 安装到手机 (Vivo iQOO 12 Pro)

### 方法 1：微信传输（最简单）
1. 把 `app-debug.apk` 发送到微信
2. 在手机微信中点击文件
3. 允许安装来自未知来源的应用
4. 点击安装

### 方法 2：USB 传输
1. 用 USB 连接手机到电脑
2. 把 APK 文件复制到手机
3. 在手机的文件管理器中点击 APK
4. 允许安装

### 方法 3：云盘
1. 上传 APK 到百度网盘 / Google Drive / iCloud
2. 在手机上下载并安装

---

## 🔧 使用 APP

### 1. 连接电脑
- 确保手机和电脑在同一 Wi-Fi
- 启动电脑端 Menu Bar App
- 打开手机 APP
- 看到 "✅ 已连接" 即可

### 2. 开始对话
- 在输入框输入消息
- 点击发送
- 会收到邓恩赐的粤语回复

### 3. 主动通知
- 当邓恩赐想你时，会主动发送消息
- 类似微信通知

## ⚠️ 常见问题

### Q: 推送时提示密码错误？
A: 需要使用 Personal Access Token，不是 GitHub 密码

### Q: 构建失败了？
A: 进入 Actions → 点击失败的构建 → 查看错误日志

### Q: APK 安装时提示风险？
A: 这是 Debug 签名的正常提示，选择"允许安装"即可

### Q: APP 无法连接电脑？
A: 检查：
1. 手机和电脑是否同一 Wi-Fi
2. 电脑端 Menu Bar App 是否运行
3. IP 地址是否正确（默认 192.168.110.254）
